# IntraAI — AI 企业内部助手平台设计文档

> 目标：从零搭建一个面向中小互联网企业的 AI 内部助手平台，核心功能为知识库/RAG、用户权限管理、管理后台。以学习为主要目的，分阶段搭建。

## 1. 整体架构

```
┌─────────────────────────────────────────────────┐
│                   用户浏览器                       │
│              Vue 3 + Ant Design Vue               │
└──────────────────────┬──────────────────────────┘
                       │ HTTP/WebSocket
┌──────────────────────▼──────────────────────────┐
│              Nginx (反向代理 + 静态资源)            │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│           FastAPI 后端 (Python)                   │
│  ┌──────────┐ ┌──────────┐ ┌─────────────────┐  │
│  │ 用户认证   │ │ 知识库管理 │ │ AI 对话引擎     │  │
│  │ JWT       │ │ 文档解析  │ │ LLM 调用 + RAG  │  │
│  └──────────┘ └──────────┘ └─────────────────┘  │
│  ┌──────────┐ ┌──────────────────────────────┐  │
│  │ 管理后台  │ │  向量检索引擎                  │  │
│  │ 统计/配置 │ │  (ChromaDB 嵌入式)            │  │
│  └──────────┘ └──────────────────────────────┘  │
└──────┬──────────────────┬───────────────────────┘
       │                  │
┌──────▼──────────┐  ┌────▼─────────────────────┐
│  MySQL 8.0      │  │  文件存储 (本地挂载卷)      │
│  业务数据        │  │  上传的文档/附件            │
└─────────────────┘  └──────────────────────────┘
```

- **MySQL**：存储用户、角色、文档元数据、会话记录等业务数据
- **ChromaDB**：存储文档切片向量，提供相似度检索（初期嵌入 Python 进程，本地持久化）
- **文件存储**：Docker 挂载卷保存原始文档

## 2. 核心模块

### 2.1 知识库/RAG 模块

- 文档上传（PDF、Word、TXT、Markdown）
- 文档解析 → 文本切片 → 向量化 → 存入 ChromaDB
- 用户提问：问题向量检索 → 取相关切片 → 拼入 Prompt → 调 LLM 生成回答
- 知识库 CRUD 管理

### 2.2 用户与权限管理模块

- 注册/登录（JWT 认证）
- 角色：管理员 / 普通用户（两级）
- 管理员：管理所有知识库和用户
- 普通用户：只能使用被授权的知识库
- 操作日志（关键操作记录）

### 2.3 管理后台模块

- 用户管理（增删改、启用/禁用）
- 知识库管理（创建、授权、查看文档列表）
- 用量统计（调用次数、Token 消耗）
- 模型配置（API Key 管理、默认模型选择）

### 前端页面

| 页面 | 功能 |
|------|------|
| 登录页 | 登录/注册 |
| 对话页 | 选择知识库 → 聊天对话（Markdown 渲染回复） |
| 知识库页 | 上传文档、管理文档列表 |
| 管理后台 | 用户管理、知识库管理、用量统计、系统设置 |

## 3. 技术栈

### 后端

| 组件 | 选择 | 说明 |
|------|------|------|
| 框架 | FastAPI (Python 3.11) | 自动生成 API 文档，异步支持好 |
| ORM | SQLAlchemy 2.0 | Python 最成熟 ORM |
| 认证 | python-jose + passlib | JWT 生成/验证，密码加密 |
| 文档解析 | PyMuPDF + python-docx | 分别处理 PDF/Word，逻辑直观 |
| 文本切片 | 自写（~50 行） | 按固定长度+重叠切分，比引入框架更易理解 |
| 向量化 | OpenAI Embedding API | 文本转向量 |
| 向量库 | ChromaDB（嵌入式） | 本地持久化，无需额外部署 |
| LLM 调用 | OpenAI SDK / httpx | 对接 GPT/Claude 等 API |

### 前端

| 组件 | 选择 | 说明 |
|------|------|------|
| 框架 | Vue 3 + Vite | 快速开发，热更新 |
| UI 库 | Ant Design Vue | 组件丰富，适合中后台 |
| 路由 | Vue Router 4 | 页面导航 |
| 状态管理 | Pinia | 轻量状态管理 |
| HTTP | Axios | API 请求 |
| Markdown 渲染 | markdown-it | AI 回复展示 |

### 部署

| 组件 | 说明 |
|------|------|
| Docker Compose | 一键启动所有服务 |
| 后端容器 | Python 3.11 + FastAPI |
| 前端容器 | Node 构建 + Nginx 托管 |
| MySQL 容器 | MySQL 8.0 |

## 5. 分阶段实施路径

以学习为主，每阶段有明确交付标准：

### 阶段 1：后端基础（FastAPI + SQLAlchemy + MySQL）
- 学习目标：理解 REST API、ORM、数据库操作
- 交付：用户注册/登录 API、JWT 认证、基础 CRUD
- 验证：用 Postman/curl 测试所有 API

### 阶段 2：前端基础（Vue 3 + Ant Design Vue）
- 学习目标：Vue 组件化开发、前后端联调
- 交付：登录页面、知识库列表页面、调用后端 API
- 验证：浏览器打开页面，能登录、查看列表

### 阶段 3：知识库功能（文档解析 + 向量化）
- 学习目标：理解文本切片、向量化、向量检索原理
- 交付：文档上传、文本切片、ChromaDB 存储与检索
- 验证：上传文档后能在 ChromaDB 中查到向量

### 阶段 4：AI 对话（LLM API + RAG 流程）
- 学习目标：Prompt Engineering、RAG 完整流程
- 交付：问题向量检索 → 拼 Prompt → 调模型 → 返回回答
- 验证：对话页问问题，AI 能基于文档回答

### 阶段 5：管理后台（权限设计 + 统计）
- 学习目标：RBAC 权限模型、数据统计设计
- 交付：用户管理、知识库授权、用量统计
- 验证：管理员能管理用户并查看统计

### 阶段 6：Docker 部署
- 学习目标：Dockerfile 编写、Compose 编排、容器网络
- 交付：docker-compose.yml 一键启动整个平台
- 验证：`docker-compose up` 后所有服务正常运行

## 6. 约束与假设

- LLM 使用外部 API（OpenAI/Claude），不在本地部署模型
- 初期用户量小，ChromaDB 嵌入式够用，后期可拆为独立服务
- 不引入消息队列，文档处理同步完成，后期量大再加 Celery
- 两级角色（管理员/用户），不做细粒度权限，够用即可
